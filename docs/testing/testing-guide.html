Path 5 — Payout on Completion

Homeowner Tradesperson

TP marks done → HO confirms (or 3-hour auto-release cron fires) → Stripe captures and transfers funds minus 10% platform fee.

Step 5.0 — KEY ACTION (TP)

Check off all checklist items → tap Mark Complete

PATCH /api/v1/jobs/:id/status
[ paste screenshot: completion screen with countdown ]
On-screen events: status pill → "Pending confirmation"; 3-hour countdown shown to HO. HO receives "Job complete — confirm & release payment" push.
▸ Request payload 
PATCH /api/v1/jobs/job_01HRA22Q9V8Q/status
{ "status": "pending_confirmation" }

→ 200 {
  "id":               "job_01HRA22Q9V8Q",
  "status":           "pending_confirmation",
  "auto_release_at":  "2026-05-18T12:32:01Z"
}
Step 5.1 — KEY ACTION (HO)

Tap Confirm & Release Payment

POST /api/v1/jobs/:id/confirm-complete
[ paste screenshot: confirmation banner + review prompt ]
On-screen events: success toast → navigate to /completion for review step. TP's Stripe Connect balance bumps within ~5 sec.
▸ Server-side capture + transfer (you won't see this in DevTools) 
POST /api/v1/jobs/job_01HRA22Q9V8Q/confirm-complete
→ server calls Stripe:
  stripe.paymentIntents.capture(pi_3PRA2K...)         // charges customer $250
  stripe.transfers.create({
    amount:      22500,                       // $250 - 10% platform fee
    currency:    "usd",
    destination: "acct_tradesperson_stripe_id"
  })

DB writes:
  UPDATE jobs SET status='completed', completed_at=now()
  INSERT INTO payments (status='completed', amount=25000, platform_fee=2500, net_payout=22500)
  UPDATE tradesperson_profiles SET jobs_completed = jobs_completed + 1
▸ Async events 
// To the tradesperson:
{
  "event":        "job.status_changed",
  "targetUserId": "usr_tradesperson_test",
  "title":        "You've been paid!",
  "body":         "$225 transferred to your Stripe Connect account.",
  "data": { "jobId": "job_01HRA22Q9V8Q", "net_payout": 22500 }
}

// Stripe webhook later (server-to-server, not in your DevTools):
POST /api/v1/stripe/webhooks
{ "type": "transfer.created", "data": { "object": { "id": "tr_..." } } }
Step 5.2 (fallback)

If HO does nothing for 3 hours, Cloud Scheduler auto-releases

cron every 30 min

Cloud Scheduler hits POST /api/v1/internal/release-expired-holds every 30 min with a shared INTERNAL_SECRET header. Any job where auto_release_at < now() gets captured and transferred just like step 5.1.
